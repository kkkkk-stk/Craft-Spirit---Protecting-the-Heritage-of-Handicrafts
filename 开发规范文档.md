# 《匠灵：守护手艺传承》开发规范文档

---

## 一、项目基本信息

- **游戏名称**：匠脉江南
- **项目类型**：2D 像素风 + 古风立绘 国风叙事解谜游戏
- **开发团队**：3人协同开发
- **开发分工**：
  - **开发者 A**：关卡一 - 浙西南畲族古寨（畲族刺绣匠灵）
  - **开发者 B**：关卡二 - 浙中东阳古木雕工坊（东阳木雕匠灵）
  - **开发者 C**：关卡三 - 浙南乐清沿海渔港（细纹刻纸匠灵）+ 主界面

## 四、关卡开发规范

### 4.1 关卡通用要素

每个关卡必须包含以下要素：

1. **场景布局**：入口点、探索点位、出口点
2. **交互物体**：非遗文物、机关、NPC
3. **记忆碎片**：4段匠人学艺记忆
4. **解谜机关**：贴合非遗工艺的解谜玩法
5. **终局任务**：复原非遗工序的交互体验
6. **通关条件**：文物收集完毕、匠灵唤醒

### 4.2 三大关卡核心玩法

| 关卡 | 核心玩法 | 解谜要素 |
|------|----------|----------|
| **关卡一** | 畲族刺绣 | 植物染材收集、纹样拼图、织布机关 |
| **关卡二** | 东阳木雕 | 榫卯拼接、分层雕刻、古建筑纹样复原 |
| **关卡三** | 细纹刻纸 | 刻纸纹样复刻、龙船灯搭建、海洋民俗图案 |

### 4.3 关卡数据接口

```csharp
// 关卡数据基类
public abstract class LevelData
{
    public string LevelName { get; set; }
    public string CraftType { get; set; }
    public string MasterName { get; set; }
    public List<string> Collectibles { get; set; }
    public List<string> MemoryClues { get; set; }
    public List<PuzzleData> Puzzles { get; set; }
    public string FinalTask { get; set; }
    
    public abstract void InitializeLevel();
    public abstract void OnLevelComplete();
}

// 解谜数据
public class PuzzleData
{
    public string PuzzleId { get; set; }
    public string PuzzleType { get; set; }
    public string[] RequiredItems { get; set; }
    public string[] RequiredMemories { get; set; }
    public string Solution { get; set; }
}
```

## 六、Git 协作规范

### 6.1 分支策略

| 分支 | 用途 | 负责人 |
|------|------|--------|
| `main` | 稳定版本，仅合并测试通过的代码 | 全员 |
| `develop` | 开发主分支，整合所有功能 | 全员 |
| `feature/level1` | 关卡一开发分支 | 开发者 A |
| `feature/level2` | 关卡二开发分支 | 开发者 B |
| `feature/level3` | 关卡三开发分支 | 开发者 C |
| `feature/main-menu` | 主界面开发分支 | 开发者 C |

### 6.2 提交规范

```
<类型>: <简要描述>

<详细描述（可选）>

类型：
- feat: 新功能
- fix: 修复
- refactor: 重构
- docs: 文档
- art: 美术资源
- audio: 音频资源
```

**示例**：
```
feat: 添加关卡一畲寨刺绣场景

- 实现畲寨村口场景布局
- 添加蓝阿婆NPC交互
- 实现植物染材收集系统
```

### 6.3 代码审查流程

1. 开发者在自己的 feature 分支上完成开发
2. 提交 PR（Pull Request）到 develop 分支
3. 其他开发者 Code Review
4. 通过后合并到 develop
5. develop 测试稳定后合并到 main

### 6.4 冲突解决

- 遇到合并冲突，优先沟通解决
- 共享资源（如通用脚本）的修改需提前同步
- 使用 Git blame 追溯修改历史

## 十、注意事项

### 10.1 开发原则

1. **非遗真实性**：所有工艺流程、纹样图案必须符合真实非遗规范
2. **文化尊重**：尊重畲族、汉族传统民俗文化
3. **团队协作**：定期同步进度，共享开发资源
4. **版本管理**：每次提交都要有清晰的提交信息

### 10.2 美术原则

1. **风格统一**：2D 像素风 + 古风立绘，保持视觉一致性
2. **色彩协调**：三关色彩区分明显但整体协调
3. **细节精致**：非遗纹样、建筑细节需精心绘制

### 10.3 测试要求

1. 每个关卡独立测试
2. 存档/读档功能测试
3. 跨平台兼容性测试
4. 性能优化测试
